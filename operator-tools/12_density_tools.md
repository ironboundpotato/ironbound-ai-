# Density Tools  
**IRONBOUND-AI Stability Architecture**  
**Document: 12_density_tools.md**

Density Tools regulate *how much* meaning is packed into each unit of text.  
Density determines cognitive load, clarity, and interpretive pressure — and is one of the operator’s most powerful stabilization levers.

Tone = emotional posture  
Precision = accuracy  
Structure = shape  
**Density = informational weight**

---

# 1. Purpose of Density Tools

Density Tools allow the operator to:

- Increase or decrease information packing  
- Control cognitive load  
- Match density to task type  
- Smooth transitions between technical and narrative modes  
- Prevent density drift during long sessions  
- Maintain the system’s “sweet spot” (6.5) when required  

Density is a dial — the operator chooses the setting.

---

# 2. Density Control Commands

Direct commands to modulate information weight:

- **“Increase density.”**  
- **“Decrease density.”**  
- **“Hold density at level X.”**  
- **“Return to density 6.5.”**  
- **“Expand this section.”**  
- **“Compress this explanation.”**  

Density changes must be explicit — never assumed.

---

# 3. Standard Density Levels

### **Density 3 — Light / Narrative**
- Loose  
- Atmospheric  
- Minimal technical load  

### **Density 5 — Balanced Exposition**
- Explanatory  
- Accessible  
- Mild abstraction  

### **Density 6.5 — The Sweet Spot**
- Conceptual clarity  
- Tight reasoning  
- High signal  
- No overload  
- Your preferred mode for most system-building  

### **Density 8 — Technical / Compressed**
- High abstraction  
- Dense meaning-per-line  
- Requires deep focus  

### **Density 10 — Maximum Compression**
- Extremely compact  
- Logic-heavy  
- Barely readable without unpacking  

The operator selects the level consciously.

---

# 4. Density Expansion Tools

Used when the model needs to unpack meaning.

Commands:

- **“Expand this idea.”**  
- **“Break this into digestible steps.”**  
- **“Unpack this concept layer by layer.”**  
- **“Increase redundancy for clarity.”**  

Expansion reduces cognitive strain and improves accessibility.

---

# 5. Density Compression Tools

Used to condense ideas into tight, high-signal format.

Commands:

- **“Condense this.”**  
- **“Remove fluff.”**  
- **“Increase signal-to-noise ratio.”**  
- **“Compress to essentials only.”**  

Compression is crucial for frameworks, summaries, and canon.

---

# 6. Density Stability Tools

Density drift occurs when:

- The model gradually becomes too verbose  
- Or too compressed  
- Or oscillates between levels unpredictably  

Stability tools:

- **“Correct density drift.”**  
- **“Re-align to density X.”**  
- **“Standardize density across this document.”**  

Density consistency = readability consistency.

---

# 7. Density Filters

Filters that shape how information is packed:

### **A. Granularity Filter**
- Macro → Meso → Micro detail  

### **B. Abstraction Filter**
- Concrete ↔ Conceptual scaling  

### **C. Brevity Filter**
- Short ↔ Moderate ↔ Extended phrasing  

These filters give the operator precision control over information weight.

---

# 8. Cross-Namespace Density Protection

Each namespace has a required density profile.

Tools:

- **“Protect namespace density.”**  
- **“Reject inappropriate density shifts.”**  
- **“Maintain canonical density for this project.”**  

Protects:

- Breaking the Pattern → ~6.5–7  
- Frameworks → 7–9  
- Fiction threads → 4–6  
- High-level theory → 8–10  

Density must stay in its world.

---

# 9. Density Audits

Used to diagnose density imbalances.

Commands:

- **“Run density audit.”**  
- **“Highlight over-dense regions.”**  
- **“Highlight under-dense regions.”**  
- **“Show density inconsistencies.”**  

Density audits catch subtle drift early.

---

# 10. Density Transitions

For switching modes mid-document:

Tools:

- **“Transition density gradually.”**  
- **“Shift from narrative to technical density.”**  
- **“Smooth the density gradient.”**  

Abrupt density changes create cognitive whiplash — transitions solve this.

---

# 11. Integration With Other Subsystems

### **Tone Tools**
Density influences emotional weight.

### **Precision Tools**
High precision often requires higher density.

### **Structure Tools**
Density must be distributed across the scaffold.

### **Session Tools**
Density drift accumulates over long sessions.

Density is a dynamic variable — not a fixed trait.
